Task-010A.3：Reminder 多筆款項清單基礎

目前已完成：

1. Reminder Editor 統一
2. Reminder 財務狀態基礎
3. 收支 UI 初步整合
4. AirportTransfer 與 Payment 金額相容處理
5. Finance Record 可透過 sourceReminderId 關聯 Reminder

目前發現新的實際需求：

一個 Reminder 不能只有一個 financialStatus。

同一個事項可能同時包含多筆款項，例如：

接機 CI836

待收：
接送費 1200

待付：
停車費 100
過路費 40

或：

帶媽媽看骨科

已支出：
掛號費 150
藥費 320

待付：
檢查費 800

因此本次建立：

「一個 Reminder 可以有多筆 Financial Items」

本次先建立資料結構、編輯 UI、清單顯示與舊資料相容。

重要：

本次不要做勾選後自動建立 Income / Expense。
本次不要做自動確認收款。
本次不要做自動確認付款。
本次不要修改 AI Parser。
本次不要修改 Groq API。
本次不要修改 Finance Store 資料結構。

────────────────────────

一、建立 Financial Item 資料型別

請先閱讀目前 Reminder 型別與 Store 結構。

使用最小修改方式新增 Financial Item。

建議型別概念：

FinancialItem {
  id: string
  title: string
  type: "receivable" | "payable"
  amount: number
  dueDate?: string
  note?: string
  completed?: boolean
  completedDate?: string
}

Reminder 新增 optional：

financialItems?: FinancialItem[]

重要：

financialItems 必須 optional。

舊 Reminder 沒有 financialItems 時必須正常運作。

不要清除 LocalStorage。

不要修改既有 LocalStorage key。

不要破壞現有 Reminder。

────────────────────────

二、本次 Financial Item 只處理預計款項

本次 Financial Item type 只需要：

receivable
payable

UI 顯示：

待收
待付

正式的：

Income
Expense

仍然使用現有 Finance Record。

不要把正式收入與正式支出存入 financialItems。

資料概念：

Financial Item
＝ 預計收款／預計付款

Finance Record
＝ 真正發生的收入／支出

兩者保持分離。

五、支援多筆款項

同一個 Reminder 可以新增多筆。

例如 AirportTransfer：

款項

☐ 待收　接送費　　 NT$ 1,200
☐ 待付　停車費　　 NT$ 100
☐ 待付　過路費　　 NT$ 40

[ ＋ 新增款項 ]

重要：

新增第二筆時不能覆蓋第一筆。

新增 payable 不能清除 receivable。

新增 receivable 不能清除 payable。

七、本次 checkbox 行為

每筆 Financial Item 左側可以顯示 checkbox 或狀態圓圈。

但是本次：

點擊 checkbox 不要直接建立 Finance Record。

如果尚未建立安全的完成流程，

點擊後可以：

A. 暫時不允許勾選，顯示 disabled 狀態

或

B. 顯示提示：
「確認收款／付款功能將於下一階段加入」

請選擇對目前程式改動最小且安全的方式。

重要：

本次絕對不要因點一下 checkbox 就重複建立 Income 或 Expense。

下一個 Task 才處理：

待收 → 確認收款 → Income

待付 → 確認付款 → Expense

十三、Reminder Card 顯示摘要

Reminder Card 不要展開所有 Financial Items。

只顯示摘要。

例如：

待收 NT$ 1,200
待付 NT$ 140

如果有實際 Finance Records：

待收 NT$ 1,200
待付 NT$ 140

收入 + NT$ 1,200
支出 - NT$ 100

不要把待收加進收入。

不要把待付加進支出。

如果金額為 0，不顯示該項摘要。

────────────────────────

十六、資料安全

請特別避免：

1. 每次 render 都新增 Financial Item
2. 每次進 EditPage 都 migration 一次
3. 重新整理後重複新增相同款項
4. 新增 Financial Item 時建立 Finance Record
5. 刪除 Financial Item 時刪除 Finance Record
6. 修改 Financial Item 時修改 Finance Dashboard 統計
7. financialItems amount 使用字串造成加總錯誤

amount 必須是 number。
