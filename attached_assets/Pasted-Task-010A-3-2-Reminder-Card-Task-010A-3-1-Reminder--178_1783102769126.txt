Task-010A.3.2：統一收支清單、逐筆完成待收待付、修正 Reminder Card 款項入口

目前 Task-010A.3 與前置財務功能已建立：

1. Reminder 可有多筆 Financial Items
2. Financial Item 支援 receivable / payable
3. Finance Record 支援 Income / Expense
4. Finance Record 可透過 sourceReminderId 關聯 Reminder
5. 舊 AirportTransfer / Payment 有相容處理

目前 UI 問題：

1. 「款項」與「實際收支」被拆成上下兩套區塊
2. 有「＋新增款項」與「＋新增收支」兩套入口
3. 使用者需要理解兩套資料概念，操作太複雜
4. 待收／待付目前缺乏簡單的逐筆完成方式
5. Reminder Card 上的記帳入口若只有小型「＋」，太小且語意不清
6. 同一 Reminder 可能同時有多筆待收、待付、收入、支出，UI 應統一顯示

本次目標：

資料底層保持分離，
但使用者 UI 統一。

不要大幅重構。
不要修改 AI Parser。
不要修改 Groq API。
不要修改 Finance Store 結構。
不要清除 LocalStorage。

UI 合併清單中只顯示一次：

仍然計入正式收支統計。

支出 - NT$ X

不要刪除 Finance Record。

不要為了 UI 去重而破壞統計。

八、防止重複建立 Finance Record

必須測試以下情況：

1. 點擊「確認已收款」一次，只建立一筆 Income
2. 點擊「確認已付款」一次，只建立一筆 Expense
3. React re-render 不會再次建立
4. 重新整理頁面不會再次建立
5. 再次進入 Editor 不會再次建立
6. completed Financial Item 不可再次執行完成流程
7. 快速連點確認按鈕不得建立兩筆相同紀錄

確認按鈕送出期間應 disabled，
避免快速重複點擊。

不要使用 useEffect 自動建立 Finance Record。

Finance Record 只能由使用者明確確認操作建立。
