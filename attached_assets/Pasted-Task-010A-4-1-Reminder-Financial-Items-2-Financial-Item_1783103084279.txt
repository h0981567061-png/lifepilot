二、總覽頁

總覽頁顯示四個主要數字：

待收總額
待付總額
本月收入
本月支出

例如：

待收
NT$ 8,500

待付
NT$ 24,360

本月收入
NT$ 35,000

本月支出
NT$ 12,500

重要：

待收不能計入收入。

待付不能計入支出。

只有 Finance Record：

Income → 收入

Expense → 支出

只有未完成 Financial Item：

receivable → 待收

payable → 待付

completed = true 的 Financial Item
不能再計入待收／待付總額。

十一、完成項目的顯示原則

收入

待付完成後：

但對應 Expense 必須出現在：

支出

✓ 已付

十五、資料來源原則

待收總額：

所有 Reminder.financialItems 中：

type = receivable
completed != true

的 amount 加總。

待付總額：

所有 Reminder.financialItems 中：

type = payable
completed != true

的 amount 加總。

收入：

Finance Records 中：

type = income

支出：

Finance Records 中：

type = expense

禁止：

待收加進收入。

待付加進支出。

completed Financial Item 再次加進待收／待付。

同一筆 Finance Record 重複計算。
