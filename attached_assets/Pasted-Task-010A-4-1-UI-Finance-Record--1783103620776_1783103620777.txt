Task-010A.4.1：補齊收入／支出紀錄的編輯與刪除功能

目前收支 UI 中：

待收／待付款項已有：
- 編輯
- 刪除

但已存在的收入／支出 Finance Record 缺少：
- 編輯
- 刪除

造成使用者若輸入錯誤金額、日期或備註後無法修正。

本次只補齊 Finance Record 的編輯與刪除操作。

不要修改 Financial Item 資料結構。
不要修改 Finance Store 核心架構。
不要修改 Reminder Editor 收支架構。
不要修改 AI Parser。
不要修改 Groq API。
不要清除 LocalStorage。

────────────────────────

一、收入與支出紀錄加入操作

所有 Income / Expense Finance Record 顯示：

收入　+ NT$ 3,500
收入日期：2026/07/20
備註：客戶尾款

[ 編輯 ] [ 刪除 ]

以及：

支出　- NT$ 100
支出日期：2026/07/20
備註：停車費

[ 編輯 ] [ 刪除 ]

請套用到：

1. Reminder Editor 的收支清單
2. 收支主頁的收入列表
3. 收支主頁的支出列表

同一筆 Finance Record 不論從哪個入口編輯，
都必須修改同一筆資料。

────────────────────────

二、編輯 Finance Record

點擊：

編輯

開啟簡潔表單：

類型
[ 收入 ] [ 支出 ]

金額
NT$ ________

日期
____ / __ / __

備註
________________

[ 儲存 ] [ 取消 ]

編輯時必須修改原本 Finance Record。

禁止：

刪除舊紀錄後新增一筆新紀錄。

禁止：

因編輯產生重複紀錄。

必須保留原 Finance Record id。

如果原本有：

sourceReminderId

編輯後必須保留，
不能因修改金額、日期、備註而失去 Reminder 關聯。

────────────────────────

三、刪除 Finance Record

點擊：

刪除

不要立即刪除。

先顯示確認：

確定刪除這筆收入紀錄？

或：

確定刪除這筆支出紀錄？

[ 確定刪除 ] [ 取消 ]

刪除後：

1. Finance Record 從 Store 移除
2. 收支統計立即更新
3. Reminder Editor 清單立即更新
4. 收支主頁立即更新
5. 不影響其他 Finance Record

────────────────────────

四、由待收／待付完成產生的紀錄

這部分必須特別處理。

如果某筆 Income 是由：

待收 → 確認已收款

產生，

或某筆 Expense 是由：

待付 → 確認已付款

產生，

編輯 Finance Record 時：

允許修改：
- 實際金額
- 實際日期
- 備註

但不要自動修改原始 Financial Item 的預計金額與預計日期。

例如：

原待收：
NT$ 3,500
預計 2026/07/20

實際收款：
NT$ 3,300
實收日期 2026/07/22

這是合法狀態。

Financial Item 保留原預計資料，
Finance Record 保存實際資料。

────────────────────────

五、刪除由待收／待付完成產生的 Finance Record

如果 Finance Record 是由完成 Financial Item 產生，
刪除時不能留下：

Financial Item = completed
但實際 Finance Record 已不存在

的不一致狀態。

請使用目前已有的 Financial Item ↔ Finance Record 關聯方式處理。

刪除這類 Finance Record 時，確認視窗顯示：

刪除這筆實際收支後，
原款項將恢復為待收。

或：

刪除這筆實際收支後，
原款項將恢復為待付。

確認後：

1. 刪除 Finance Record
2. 對應 Financial Item：
   completed = false
3. 清除 completedDate
4. 該項目重新出現在待收或待付清單
5. 待收／待付總額立即更新

如果目前無法可靠找到對應 Financial Item，
不要使用金額或日期猜測配對。

只能使用既有唯一關聯欄位。

若目前尚無可靠唯一關聯，
請停止這一小段功能並回報，
不要自行以 amount + date 配對。

────────────────────────

六、直接新增的收入／支出

如果 Income / Expense 是使用者直接透過：

＋ 新增款項
→ 收入

或：

＋ 新增款項
→ 支出

建立，

刪除時只刪除 Finance Record。

不要建立或恢復任何 Financial Item。

────────────────────────

七、手機版操作

目前畫面右側的：

編輯
刪除

文字偏小。

請改成手機容易點擊的操作方式。

可以使用：

[ 編輯 ] [ 刪除 ]

或清楚的更多操作選單。

但不要使用過小的 icon-only button。

刪除操作必須與主要內容保持適當距離，
避免誤觸。

────────────────────────

八、測試

測試 1：
直接新增收入 3500。
編輯成 3800。
預期同一筆紀錄更新，不產生第二筆。

測試 2：
直接新增支出 100。
修改日期與備註。
預期 id 不變，資料更新。

測試 3：
刪除直接新增的收入。
預期只刪除 Finance Record。

測試 4：
待收 3500 → 確認收款 → 產生 Income。
修改實收金額為 3300。
預期原待收預計金額仍為 3500，
實際 Income 為 3300。

測試 5：
刪除由待收完成產生的 Income。
預期原 Financial Item 恢復為待收。

測試 6：
刪除由待付完成產生的 Expense。
預期原 Financial Item 恢復為待付。

測試 7：
從 Reminder Editor 修改 Finance Record。
進入收支主頁後資料同步更新。

測試 8：
從收支主頁刪除 Finance Record。
返回 Reminder Editor 後同步更新。
