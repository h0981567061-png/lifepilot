修正 AI 智慧貼上的「日期區間解析」問題。

目前案例：

桌球7/20-7/24
平興國小
（時間14點-17點）

目前解析結果：

標題：桌球
日期：2026/07/20
時間：14:00～17:00
地點：平興國小
備註：7/20-7/24

這是錯誤的。

正確結果應為：

標題：桌球

日期模式：
日期區間

開始日期：
2026/07/20

結束日期：
2026/07/24

時間模式：
時間區間

開始時間：
14:00

結束時間：
17:00

地點：
平興國小

備註：
空白

────────────────────────

一、日期欄位增加日期模式

Reminder 日期需要支援：

dateMode:
single
range

單日：

dateMode = single
startDate = 2026-07-20
endDate = null

日期區間：

dateMode = range
startDate = 2026-07-20
endDate = 2026-07-24

如果目前 Reminder 已有 date 欄位，
請做向下相容 migration。

不要破壞舊 Reminder。

舊資料只有 date 時：

視為：

dateMode = single
startDate = 原 date

────────────────────────

二、新增／編輯 UI

日期區域增加：

[ 單日 ] [ 日期區間 ]

選擇單日：

日期
[ 2026/07/20 ]

選擇日期區間：

開始日期
[ 2026/07/20 ]

結束日期
[ 2026/07/24 ]

時間區域維持現有：

[ 全天 ] [ 單一時間 ] [ 時間區間 ]

因此本案例應顯示：

日期：
[ 日期區間 ]

2026/07/20 ～ 2026/07/24

時間：
[ 時間區間 ]

14:00 ～ 17:00

────────────────────────

三、AI Parser 日期區間辨識

需要辨識常見格式：

7/20-7/24
7/20～7/24
7/20~7/24
7月20日到7月24日
7月20日至7月24日
7/20 到 7/24
7/20 至 7/24

解析為：

dateMode = range
startDate
endDate

不要只取第一天。

不要把剩餘日期文字放入備註。

────────────────────────

四、年份處理

如果輸入沒有年份：

依目前系統既有年份推斷規則處理。

例如目前解析環境判定為 2026：

7/20-7/24

→

2026/07/20
至
2026/07/24

如果跨年：

12/30-1/2

應解析為：

2026/12/30
至
2027/01/02

不要解析成結束日期早於開始日期。

────────────────────────

五、月份行事曆顯示

日期區間 Reminder 必須在區間內每一天可被月份行事曆查詢到。

例如：

桌球
7/20～7/24
14:00～17:00

月份行事曆：

7/20 顯示桌球
7/21 顯示桌球
7/22 顯示桌球
7/23 顯示桌球
7/24 顯示桌球

但資料庫中仍然只有一筆 Reminder。

禁止為五天建立五筆重複 Reminder。

月曆顯示使用 derived date expansion。

七、避免與時間區間混淆

本案例同時包含：

日期區間：
7/20～7/24

時間區間：
14:00～17:00

Parser 必須分別處理。

不得將：

7/20-7/24

誤判成時間或備註。

不得將：

14點-17點

誤判成日期。
