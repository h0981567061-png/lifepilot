Task-011A：建立「工作資料板」V1 基礎架構

目前專案已有 Reminder、Category、Finance 等功能。

現在新增「工作資料板」功能。

本次只建立工作資料板的基礎管理能力。

不要修改 AI Parser。
不要修改 Groq API。
不要修改 Reminder Type。
不要移除 AirportTransfer。
不要遷移舊資料。
不要修改 Finance Store。
不要修改 FinancialItem。
不要清除 LocalStorage。
不要大幅重構 App.tsx。

────────────────────────

一、功能定位

「工作資料板」不是事項類型。

它代表使用者的一個實際工作身份或工作來源。

例如：

A 車行
模板：機場接送

B 車行
模板：機場接送

個人接案
模板：一般工作

同一位使用者可以建立多個工作資料板。

允許：

A 車行 → 機場接送
B 車行 → 機場接送

也就是多個工作資料板可以使用相同模板。

────────────────────────

二、建立 WorkProfile 資料模型

建立最小資料模型：

WorkProfile

id
name
templateType
enabled
note
createdAt
updatedAt

templateType V1 先支援：

general_work
airport_transfer

例如：

{
  id: "work_xxx",
  name: "A 車行",
  templateType: "airport_transfer",
  enabled: true,
  note: "",
  createdAt: "...",
  updatedAt: "..."
}

重要：

WorkProfile 只保存工作資料板本身的設定。

不要在 WorkProfile 裡保存單次行程資料，例如：

航班號碼
乘客姓名
單次出發地
單次目的地
單次金額

這些未來屬於單次 Reminder / Template Data。

────────────────────────

三、建立 WorkProfile Store

建立獨立的 WorkProfile Store。

至少提供：

getWorkProfiles()
getWorkProfileById(id)
createWorkProfile(data)
updateWorkProfile(id, data)
deleteWorkProfile(id)

使用目前專案既有的持久化方式。

如果目前專案使用 LocalStorage，
WorkProfile V1 也使用相同風格。

不要修改其他 Store。

不要與 Category Store 混用。

────────────────────────

四、「我的」頁加入工作資料板入口

在「我的」頁新增：

工作資料板

副文字：

管理不同工作與工作模板

點擊後進入工作資料板管理頁。

不要把所有工作資料直接塞在「我的」首頁。

────────────────────────

五、工作資料板管理頁

頁面標題：

我的工作

顯示所有 WorkProfile。

例如：

A 車行
機場接送

B 車行
機場接送

個人接案
一般工作

底部或適當位置：

＋ 新增工作

每張工作卡片提供：

編輯
刪除

手機版按鈕必須容易點擊。

不要使用過小的 icon-only button。

────────────────────────

六、新增工作

點擊：

＋ 新增工作

顯示：

工作名稱

________________

工作模板

[ 一般工作 ▼ ]

可選：

一般工作
機場接送

備註（選填）

________________

啟用此工作

[ 開關 ]

[ 儲存 ] [ 取消 ]

例如：

工作名稱：
A 車行

工作模板：
機場接送

儲存後建立 WorkProfile。

────────────────────────

七、編輯工作

允許修改：

工作名稱
工作模板
備註
enabled

編輯時修改原 WorkProfile。

必須保留原 id。

不要刪除後重新建立。

────────────────────────

八、刪除工作

點擊刪除時必須先確認：

確定刪除這個工作資料板？

[ 確定刪除 ] [ 取消 ]

目前 V1 尚未正式將 Reminder 關聯 WorkProfile，

所以本次只刪除 WorkProfile。

未來建立 Reminder 關聯後，
再加入關聯資料保護規則。

────────────────────────

內部：

UI 顯示：

內部：
