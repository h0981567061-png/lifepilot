Task：首頁／今日總覽 V1

目前系統已有：

- Reminder Store
- 提醒事項列表
- 月份行事曆
- FinancialItem／統一收支資料
- 收支月份瀏覽
- WorkProfile／多工作資料
- 一般事項
- 機場接送專屬事項
- AI 智慧貼上
- 手動新增事項

本次目標：

建立「首頁／今日總覽」。

首頁不是新的資料系統，
而是現有資料的總覽入口。

首頁資料來源：

Reminder Store
FinancialItem 正式資料來源
WorkProfile 正式資料來源

禁止建立：

Home Store
Dashboard Store
Today Store
Home Reminder 副本
Home Finance 副本

首頁所有內容都必須是 derived data。

────────────────────────

1. 日期與問候區

4. 需要處理
5. 收支待辦摘要

二、今日事項

「今日事項」只讀取 Reminder Store。

條件：

Reminder 日期 = 今天
且
未完成

依時間排序。

排序規則：

有時間的事項：
依時間早到晚排序。

全天事項：
放在時間事項之前或依目前 UI 規則一致處理。

沒有時間：
放在有時間事項之後。

每張卡片顯示：

時間
標題
類型
地點（有資料才顯示）
工作名稱（有關聯才顯示）
重要收支狀態（有資料才顯示）

不要顯示空白欄位。

────────────────────────

三、機場接送首頁摘要

如果事項為機場接送類型：

首頁卡片可以顯示：

22:05
機場接送

CI836 · 接機
桃園機場 → 新店區

待收 NT$1,200

只顯示有資料的欄位。

不要顯示：

空白航班
空白地區
空白乘客
空白車型

首頁只顯示摘要。

完整資料仍進入 Reminder 詳情頁查看。

────────────────────────

四、即將到來

顯示今天之後最近的未完成 Reminder。

V1 建議範圍：

未來 7 天。

依日期與時間排序。

最多先顯示 5 筆。

如果超過：

顯示：

查看全部提醒事項

點擊後進入現有提醒事項頁。

不要建立 Upcoming Store。

────────────────────────

五、需要處理

本區顯示真正需要使用者注意的資料。

V1 包含：

1. 已逾期但未完成的 Reminder
2. 已到期但仍為待收的 FinancialItem
3. 已到期但仍為待付的 FinancialItem

排序優先順序：

逾期時間最久
→ 最近需要處理

顯示範例：

逾期 3 天
信用卡帳單
待付 NT$23,560

或：

逾期 1 天
客戶尾款
待收 NT$3,500

不要自行複製 Reminder 或 FinancialItem。

────────────────────────

六、首頁摘要數字

首頁上方顯示簡單摘要：

今日事項數量
待處理數量
待收總額
待付總額

計算方式：

今日事項數量：
今天且未完成的 Reminder 數量。

待處理數量：
逾期未完成 Reminder 與符合待處理條件的收支項目數量。

待收總額：
目前正式 FinancialItem 中符合待收狀態的總額。

待付總額：
目前正式 FinancialItem 中符合待付狀態的總額。

注意：

如果 Reminder 與 FinancialItem 是關聯資料，
待處理數量不得因同一件事情重複計算兩次。

先檢查目前關聯方式，
使用正式 ID 關聯去重。

不要使用標題文字比對去重。

────────────────────────

七、首頁卡片操作

點擊 Reminder 卡片：

開啟現有 Reminder 詳情或編輯流程。

不要建立 Home Reminder Editor。

點擊收支項目：

開啟現有收支詳情或編輯流程。

不要建立 Home Finance Editor。

從原流程修改後返回首頁：

首頁必須立即更新。

────────────────────────

八、完成事項同步

如果使用者在 Reminder 正式流程完成事項：

首頁今日事項立即消失或更新。

提醒事項列表同步更新。

月份行事曆同步更新。

不得建立首頁自己的完成狀態。

────────────────────────

九、收支狀態同步

如果：

待收 → 已收

首頁的：

待收總額
需要處理
收支待辦

必須立即更新。

如果：

待付 → 已付

首頁的：

待付總額
需要處理
收支待辦

必須立即更新。

首頁不得自行保存收支狀態。

────────────────────────

十、首頁空狀態

如果今天沒有事項：

顯示：

今天沒有待辦事項

[ ＋ 新增事項 ]

點擊後進入現有新增事項入口。

仍保留：

AI 智慧貼上
手動新增

不要建立首頁專用新增表單。

如果沒有待處理：

顯示：

目前沒有逾期或待處理項目

如果沒有待收待付：

顯示：

目前沒有待處理收支

────────────────────────

十一、首頁快速新增

首頁提供清楚的：

＋ 新增事項

點擊後進入現有新增流程。

流程仍然是：

新增事項
├── AI 智慧貼上
└── 手動新增

不要讓首頁直接建立不完整 Reminder。

────────────────────────

十二、工作資料顯示原則

首頁可以顯示 Reminder 關聯的工作名稱。

例如：

木工
機場接送
教學

但不要把 WorkProfile 的所有資料直接塞進首頁。

工作資料只作為：

標籤
識別
摘要資訊來源

完整工作資料仍在「我的 → 工作」管理。

────────────────────────

十三、日期規則

今日事項：

使用 Reminder 正式日期欄位。

即將到來：

使用 Reminder 正式日期欄位。

收支到期判定：

使用 FinancialItem 正式日期／到期日期欄位。

禁止：

從標題解析日期
從備註解析日期
從 AI 原始文字重新推算日期
使用 createdAt 取代正式日期

如果目前 FinancialItem 沒有明確到期日期，
請先使用現有最合理的正式日期欄位，
並在完成報告中說明。

不要自行建立互相衝突的新日期系統。

────────────────────────

十四、手機版 UI

首頁以手機版優先。

要求：

資訊不能太擁擠。
摘要卡片數字清楚。
今日事項容易掃讀。
時間欄位清楚。
金額清楚。
狀態顏色一致。
長標題不能破版。
長地點要截斷或換行。
Bottom Navigation 不得遮住最後內容。
不得水平捲動。

摘要區不要塞太多大型卡片。

可以使用 2 × 2 摘要排列：

今日事項
待處理
待收
待付

────────────────────────

十五、顏色規則

沿用目前 App Design System。

建議：

一般資訊：藍色系
今日事項：主題色
逾期：紅色系
待收：綠色或青綠色系
待付：橘色或紅橘色系

不要新增與目前 App 完全不一致的設計語言。

────────────────────────

十六、測試案例

測試 1：

今天有 3 筆未完成 Reminder。

預期：
首頁今日事項顯示 3 筆。
摘要顯示今日 3。

────────────────

測試 2：

今天事項完成 1 筆。

預期：
首頁立即剩 2 筆。
提醒列表與月曆同步。

────────────────

測試 3：

未來 7 天有 8 筆 Reminder。

預期：
首頁即將到來只顯示前 5 筆。
可進入查看全部提醒事項。

────────────────

測試 4：

有逾期 Reminder。

預期：
出現在需要處理區。

────────────────

測試 5：

有待收 NT$3,500。

預期：
首頁摘要與收支待辦正確顯示。

────────────────

測試 6：

待收轉已收。

預期：
待收總額立即扣除。
需要處理區同步更新。

────────────────

測試 7：

有待付 NT$1,200。

預期：
首頁正確顯示待付總額。

────────────────

測試 8：

修改 Reminder 日期。

今天 → 明天。

預期：
從今日事項消失。
進入即將到來區。

────────────────

測試 9：

刪除 Reminder。

預期：
首頁、提醒列表、月份月曆同步消失。

────────────────

測試 10：

機場接送 Reminder。

預期：
首頁顯示時間、方向、航班、路線摘要與收支狀態。
只顯示有資料欄位。

────────────────

測試 11：

今天完全沒有事項。

預期：
顯示正確空狀態與新增事項入口。

────────────────

測試 12：

手機寬度測試。

預期：
無水平捲動。
卡片不破版。
Bottom Navigation 不遮內容。
