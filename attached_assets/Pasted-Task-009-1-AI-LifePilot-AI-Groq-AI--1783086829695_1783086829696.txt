────────────────

一、修正時間格式錯誤

目前測試 Airport Transfer 時發現：

原始輸入：

21:15 / 接機 / CI173 / 中山 / 正七以上 / 1000 元

但建立提醒事項後，
部分畫面可能顯示成：

21::15

請檢查完整資料流程：

Parser Result
→ Preview Card
→ 建立所選
→ Reminder Storage
→ RemindersPage
→ EditPage

要求：

1. 時間統一保存為 HH:mm。

例如：

21:15
09:00
23:55

2. 不可以出現：

21::15
9::00
21:15:
或其他錯誤格式。

3. 建立共用 normalizeTime() 函式，
放在適合的共用 utils 檔案中。

4. 不要只在 UI 顯示時修正，
資料寫入 Reminder Storage 前也必須正規化。

5. normalizeTime() 至少支援：

21:15 → 21:15
2115 → 21:15
09:00 → 09:00
9:00 → 09:00
21點15分 → 21:15
晚上9點15分 → 21:15

6. 如果時間無法可靠解析，
不要猜測，保留 null 或原本安全處理方式。

────────────────

二、缺少日期時提供補日期功能

目前測試資料：

21:15 / 接機 / CI173 / 中山 / 正七以上 / 1000 元
21:00 / 接機 / BR68 / 大安 / 小車 / 900 元
21:30 / 接機 / VN0578 / 北投 / 小車 / 900 元
23:55 / 送機 / 桃園八德 / 正七以上 / 900 元 / 備註兒童椅

因為原文沒有日期，
分析結果卡片必須清楚顯示：

⚠ 尚未設定日期

並提供：

「選擇日期」

使用者可以直接在每一張分析結果卡片中補上或修改日期。

要求：

1. 已有日期的事項顯示目前日期。
2. 沒有日期的事項顯示缺少日期警告。
3. 每一張卡片都可以單獨選擇或修改日期。
4. 修改後 Preview Card 必須立即更新。
5. 建立所選後，日期必須正確保存到 Reminder Storage。
6. 提醒事項列表必須依日期正確分類。
7. 編輯頁仍可以再次修改日期。
8. 不要擅自猜測日期。
9. 不要自動套用今天日期。

────────────────

三、新增「全部套用日期」功能

Airport Transfer 等類型常常一次貼入多筆同一天資料。

請在分析結果區加入：

「全部套用日期」

或語意清楚的同等按鈕。

操作方式：

1. 使用者點擊「全部套用日期」。
2. 開啟日期選擇器。
3. 選擇日期。
4. 將日期套用到目前所有「已勾選」事項。

如果部分已勾選事項原本已有日期，
顯示確認選項：

- 取消
- 只套用無日期事項
- 全部覆蓋

範例：

找到 4 個事項
已選取 4 / 4

使用者選擇：

2026/07/05

結果：

4 個已選取事項全部套用 2026/07/05。

如果使用者只勾選 2 筆，
則只套用到這 2 筆。

────────────────

四、建立所選前的日期檢查

當使用者按下「建立所選」時：

如果已勾選事項中仍有缺少日期的資料，
顯示：

「有 X 個事項尚未設定日期，是否仍要建立？」

提供：

- 返回設定日期
- 仍然建立

要求：

1. 選擇「返回設定日期」時，不建立任何資料，保留目前分析結果與勾選狀態。
2. 選擇「仍然建立」時，允許建立，並放入提醒事項的「無日期」分類。
3. 不要因為缺少日期而刪除事項。
4. 不要自動補今天日期。

3 個 Airport Transfer events

如果 AI 回傳的每筆 event 已有正確日期，
前端直接使用 AI 結果。

不要在前端自行猜測原文日期語意。

規則 Parser fallback 如果已有安全可靠的共用日期邏輯則保留；
如果沒有，不要為了本次任務大幅重寫 Parser。

七、輸入框 UX 優化

目前 textarea 有實際預填的測試資料。

請修改為：

1. textarea 預設 value 為空字串。
2. 不要自動放入真正測試資料。
3. 改用淡色 placeholder 顯示範例。
4. 使用者開始輸入或貼上文字後，placeholder 自動消失。
5. placeholder 不可被 Analyze 當成真正資料。

Placeholder 內容：

例如：

明天下午3點到桃園榮總看骨科，
回家前買牛奶、雞蛋和衛生紙，

接送工作、付款資訊或其他生活事項。

輸入為空時：

Analyze 按鈕應 disabled，
或點擊時顯示清楚提示。
