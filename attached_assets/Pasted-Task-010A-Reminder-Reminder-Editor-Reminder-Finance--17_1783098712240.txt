Task-010A：建立 Reminder 財務狀態與待收／待付基礎

目前 Reminder Editor 已完成統一。

下一步建立 Reminder 與 Finance 之間的財務狀態基礎。

本次目標：

1. Reminder 可以設定財務狀態
2. 支援無財務關聯、待收、待付
3. 顯示預計金額
4. 顯示財務到期日
5. AI 建立與手動新增的 Reminder 使用相同資料邏輯
6. 待收／待付不能算入正式 Finance 收支
7. 相關收支區塊移到提醒設定上方

重要：
本次只建立財務狀態基礎。
不要做確認收款。
不要做確認付款。
不要自動建立 Finance Record。
不要修改 Groq API。
不要修改 AI Prompt。
不要修改 Parser 判斷邏輯。

────────────────

一、Reminder 財務狀態

請在 Reminder 資料中以 optional 欄位加入財務資訊。

請依目前專案實際型別結構選擇最小修改方式。

需要支援：

financialStatus:

none
receivable
payable

語意：

none：
無財務關聯

receivable：
待收

payable：
待付

另外支援：

expectedAmount?: number

financialDueDate?: string

重要：

新欄位必須 optional。

舊 Reminder 沒有這些欄位時，
必須正常顯示與編輯。

不要進行破壞性 migration。
不要清除 LocalStorage。

────────────────

二、Reminder Editor 財務區塊

所有 Reminder Editor 都可以使用財務狀態。

在基本資料與 Type 專屬欄位之後，
提醒設定之前，
新增：

財務資訊

狀態：

無
待收
待付

如果選擇：

無

隱藏：

預計金額
財務到期日

如果選擇：

待收

顯示：

預計金額
預計收款日（選填）

如果選擇：

待付

顯示：

預計金額
付款期限（選填）

三、Payment Reminder 的處理

目前 Payment 例如：

信用卡繳費
截止日期：2026/08/05
金額：23560

這筆資料的財務語意應為：

financialStatus = payable

expectedAmount = 23560

financialDueDate = 2026/08/05

UI 顯示：

財務資訊

待付

預計金額
NT$ 23,560

付款期限
2026/08/05

重要：

不要因為 Payment 有 amount，
就建立 Expense Finance Record。

23560 現階段是：

待付金額

不是：

已支出 23560

────────────────

四、AirportTransfer Reminder 的處理

例如：

CI173 接機
接送費用 1000

財務狀態可以設定為：

待收

預計金額：

1000

UI 顯示：

財務資訊

待收

預計金額
NT$ 1,000

重要：

AirportTransfer 的接送費用不能直接顯示為正式收入。

在沒有 Finance Record 前：

不能顯示：

收入 + NT$ 1,000

只能顯示：

待收 NT$ 1,000

六、舊資料相容

目前可能已有：

Payment.amount
Payment.dueDate

AirportTransfer.amount

請不要刪除舊欄位。

如果舊 Payment Reminder 沒有 financialStatus，
但有 amount 與 dueDate，

Editor 可以用相容方式顯示為：

待付
預計金額 = amount
付款期限 = dueDate

如果舊 AirportTransfer 沒有 financialStatus，
不要直接假設已收入。

可以保留原本接送費用顯示，
但正式 Finance Summary 不得將它算成收入。

避免一次進行大量資料 migration。

九、Reminder Card 顯示

如果 Reminder 為：

receivable

且尚未完成正式收款：

顯示：

待收 NT$ 1,000

如果 Reminder 為：

payable

顯示：

待付 NT$ 23,560

待收與待付使用清楚但不過度醒目的狀態色。

重要：

待收不能加進收入總額。

待付不能加進支出總額。
